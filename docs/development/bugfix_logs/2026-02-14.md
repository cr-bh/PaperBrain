### 问题 1: SSL 证书验证失败导致论文导入失败

**问题描述**：
- 从 Auto-Scholar 收藏列表导入论文时报错
- 错误信息：`HTTPSConnectionPool(host='aigc.sankuai.com', port=443): Max retries exceeded`
- 具体错误：`SSLZeroReturnError: TLS/SSL connection has been closed (EOF)`

**影响范围**：
- 论文导入功能完全无法使用
- 所有需要调用 LLM API 的功能都会失败

**根本原因**：
- 公司内部 API (`aigc.sankuai.com`) 的 SSL 证书验证问题
- `llm_service.py` 没有使用自定义 SSL 适配器
- `doubao_service.py` 之前已经修复过相同问题，但 `llm_service.py` 遗漏了

**为什么昨天可以今天不行**：
- 可能是网络环境变化（VPN、代理切换）
- 可能是 API 服务端证书更新
- 可能是系统 SSL 库更新
- **不是今天的代码迭代导致的**（今天只修改了 README 图片显示）

**解决方案**：

在 `services/llm_service.py` 中添加了与 `doubao_service.py` 相同的 SSL 解决方案：

1. **添加 SSLAdapter 类**：
```python
import urllib3
from requests.adapters import HTTPAdapter
from urllib3.util.ssl_ import create_urllib3_context

urllib3.disable_warnings(urllib3.exceptions.InsecureRequestWarning)

class SSLAdapter(HTTPAdapter):
    """自定义 SSL 适配器，解决公司 API 的 SSL 证书验证问题"""

    def init_poolmanager(self, *args, **kwargs):
        context = create_urllib3_context()
        # 降低 SSL 安全级别以兼容公司内部 API
        context.set_ciphers('DEFAULT@SECLEVEL=1')
        kwargs['ssl_context'] = context
        return super().init_poolmanager(*args, **kwargs)
```

2. **使用 requests.Session**：
```python
def __init__(self):
    # ...
    # 创建一个持久的 Session，使用自定义 SSL 配置
    self.session = requests.Session()
    self.session.mount('https://', SSLAdapter())
    # 禁用 SSL 验证（因为公司 API 的证书有问题）
    self.session.verify = False
```

3. **修改 API 调用**：
```python
# 从 requests.post() 改为 self.session.post()
response = self.session.post(self.api_url, headers=headers, data=json_data, timeout=120)
```

**修改文件**：
- `services/llm_service.py`

**测试结果**：
- ✅ 论文导入功能恢复正常
- ✅ 所有 LLM API 调用正常

**Git 提交**：
- Commit: `0efe21f`
- Message: "Fix SSL certificate error for LLM service"

### 问题 2: "昨天到目前"模式无法抓取论文

**问题描述**：
- 选择"昨天到目前"抓取模式时，立即结束，没有抓取到任何论文
- 选择"自定义时间段"时可以正常抓取

**影响范围**：
- "昨天到目前"快捷模式完全无法使用
- 用户必须手动选择日期范围

**根本原因**：

1. **时区和更新时间问题**：
   - Arxiv 在美国东部时间晚上 8 点更新（北京时间早上 8-9 点）
   - 如果在下午或晚上测试，"今天"的论文还没发布
   - "昨天"的论文可能已经被之前抓取过了

2. **去重过滤导致空结果**：
   - 代码会检查论文是否已存在数据库中
   - 如果昨天的论文都已经抓取过，会被全部过滤掉
   - 导致返回 0 篇论文，流程立即结束

3. **为什么自定义时间段可以**：
   - 更长的时间范围（如 7 天）包含更多未抓取的论文
   - 所以能正常工作

**代码分析**：

原代码（`ui/auto_scholar.py` 第 178-188 行）：
```python
else:  # 昨天到目前模式
    # 昨天 00:00 到今天 23:59
    yesterday = datetime.now() - timedelta(days=1)
    start_dt = datetime.combine(yesterday.date(), datetime.min.time())
    end_dt = datetime.combine(datetime.now().date(), datetime.max.time())
```

问题：只查询 1 天的数据，容易被去重过滤完。

**解决方案**：

**方案 1: 扩大时间范围（已采用）**

修改 `ui/auto_scholar.py`：
```python
else:  # 昨天到目前模式
    # 最近2天：前天 00:00 到今天 23:59
    # 这样可以确保抓到论文（考虑到 Arxiv 更新时间和时区差异）
    two_days_ago = datetime.now() - timedelta(days=2)
    start_dt = datetime.combine(two_days_ago.date(), datetime.min.time())
    end_dt = datetime.combine(datetime.now().date(), datetime.max.time())
```

**优势**：
- ✅ 扩大时间范围，确保能抓到论文
- ✅ 考虑时区差异和 Arxiv 更新时间
- ✅ 即使昨天的论文被抓取过，前天的可能还有新的

**方案 2: 添加去重计数器（辅助优化）**

修改 `services/arxiv_crawler.py`：
```python
papers = []
duplicates = 0  # 记录去重数量

for result in search.results():
    # ...
    if db_manager.get_arxiv_paper_by_arxiv_id(result.entry_id.split('/')[-1]):
        duplicates += 1

    # ...

if duplicates > 0:
    print(f"✓ 成功抓取 {len(papers)} 篇新论文（已过滤 {duplicates} 篇重复论文）")
else:
    print(f"✓ 成功抓取 {len(papers)} 篇新论文")
```

**优势**：
- ✅ 显示有多少论文被去重过滤
- ✅ 帮助用户理解为什么结果较少
- ✅ 便于调试和问题诊断

**修改文件**：
- `ui/auto_scholar.py` - 修改时间范围逻辑
- `services/arxiv_crawler.py` - 添加去重计数器

**测试结果**：
- ✅ "昨天到目前"模式可以正常抓取论文
- ✅ 显示去重信息，用户体验更好

## 🔍 技术细节

### SSL 证书问题的技术原理

**问题**：
- OpenSSL 3.0+ 默认使用 `SECLEVEL=2`
- 公司内部 API 的证书可能使用较旧的加密算法
- 导致 SSL 握手失败

**解决方案**：
- 降低 SSL 安全级别到 `SECLEVEL=1`
- 禁用 SSL 证书验证（仅用于内部 API）
- 使用自定义 SSL 上下文

**安全性考虑**：
- ⚠️ 仅用于公司内部 API
- ⚠️ 不建议用于公开的外部 API
- ✅ 已禁用 SSL 警告，避免日志污染

---

### 时区和 Arxiv 更新机制

**Arxiv 更新时间**：
- 美国东部时间（EST）晚上 8:00 PM
- 北京时间（CST）早上 8:00 AM（冬令时）或 9:00 AM（夏令时）

**论文发布流程**：
1. 作者提交论文
2. Arxiv 审核（通常 1-2 天）
3. 在固定时间批量发布
4. 论文的 `submittedDate` 是提交日期，不是发布日期

**时间范围选择策略**：
- 查询"昨天"的论文，实际可能是前天提交的
- 查询"今天"的论文，可能还没发布
- 因此使用 2 天的时间范围更保险
